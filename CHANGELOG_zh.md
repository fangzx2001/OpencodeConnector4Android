# 更新日志

OConnector 的所有重要变更都会记录在此文件中。

## [1.1.0] - 2026-05-05

### 新增

- **多项目自动发现** — APP 现在会并行查询所有已知的 OpenCode 项目，合并展示每个项目的会话。无论 `opencode serve` 从哪个目录启动都能正常工作。

### 修复

- **暗色模式** — 之前完全无法使用（主题从未被应用到 Activity）。新增手动 🌙/☀️ 切换按钮，设置保存在 DataStore 中。
- **Todo 按钮可见性** — Todo 按钮在列表为空时隐藏。现在始终可见（有任务时显示角标，无任务时显示普通图标）。同时新增 `message.part.completed` SSE 事件处理，实现 Todo 实时更新。
- **工具调用渲染** — 连续的工具调用会相互覆盖，因为 `putSegment()` 仅按类型匹配。新增 `callID` 追踪，确保每个工具调用拥有独立的渲染段。
- **内存溢出崩溃** — 加载长会话时出现 108MB 内存分配崩溃。消息 API 新增 `?limit=50` 参数。移除了会导致 JSON 转义序列损坏的 `truncateLargeJsonStrings()`（此函数是消息/会话"消失"的根本原因）。
- **会话排序** — 会话按创建时间排序。现在改为按最后更新时间排序。
- **状态栏颜色** — 状态栏和导航栏颜色现在在亮色和暗色模式下都与界面颜色一致。
- **SSE 流式输出卡顿** — 流式输出有时会随机卡住不动，需要手动刷新。根因：`channelFlow` 中的 `trySend()` 在主线程繁忙时会静默丢弃 SSE 事件。已替换为 `send()` 提供正确的背压机制。同时将逐条日志改为每 50 条输出一次，减少主线程 JNI 开销。

### 变更

- 服务器启动脚本不再需要从特定目录启动。
- `listSessions()` 新增可选的 `scope` 参数用于服务端项目过滤。

## [1.0.0] - 2026-04-30

### 新增

- 首个正式版本。
- 通过 IP + 端口连接 OpenCode 服务器。
- 浏览项目和会话列表。
- 实时 AI 聊天，SSE 流式输出。
- 工具调用渲染（思考气泡、工具摘要）。
- Todo 任务面板浮层。
- Agent 选择器（切换不同 AI Agent）。
- 会话 Fork / 删除。
- 中英双语切换。
- 密码加密存储。
- SSE 自动重连（指数退避）。
