# 更新日志

OConnector 的所有重要变更都会记录在此文件中。

## [1.2.0] - 2026-05-11

### 新增

- **侧边面板** — 在屏幕任意位置左滑打开文件浏览器和模型信息面板。浏览项目目录、查看模型/供应商详情、追踪上下文窗口用量。
- **文件预览** — 在文件浏览器中点击 `.md` 或 `.txt` 文件可直接预览内容，无需离开聊天。使用 `GET /file/content?path=` 接口。
- **Todo 任务面板** — 浮层面板展示 AI 任务进度。工具栏显示数字徽标。全部完成时自动关闭并推送通知。
- **询问/确认气泡** — 当 AI 需要权限确认或提出问题时，消息列表和输入栏之间显示内联气泡。权限气泡有「允许一次」/「始终允许」/「拒绝」。问题气泡有可选项、自定义文本输入、「提交」/「忽略」。AI 等待回复时输入栏被禁用。
- **逐题回答** — 当 AI 一次提出多个问题时，逐题展示，支持「上一步」/「下一步」导航。显示进度条，所有回答收集后一次性提交。
- **逐个权限确认** — 多个权限请求自动排队，逐个呈现。确认/拒绝一个后自动弹出下一个。
- **多选题支持** — 支持带 `multiple: true` 标记的问题，可同时选择多个选项。
- **撤销/重做** — 撤销上一条用户消息（服务端软隐藏消息 + 回滚文件变更）。重做恢复已撤销的消息。工具栏按钮按需显示。被撤销的文本自动恢复到输入框，方便重新编辑。
- **逐条消息切换模型** — 每条消息可独立选择 AI 模型，使用嵌套的 `{ providerID, modelID }` 选择器。发送时携带 `ModelRef`，服务端精确使用所选模型。
- **上下文用量显示** — 在输入栏模型选择器旁显示当前上下文 token 消耗（如 "32K"），数据取自最新 assistant 消息的 `input` token。
- **手动检查更新按钮** — 连接页面新增刷新按钮，可随时手动触发更新检查，不再仅依赖启动时自动检查。
- **更新检查镜像源** — 同时检查 GitHub + `gh-proxy.com` CDN 镜像，Channel 竞速。任一源响应即检测到更新，不再受 GitHub 连接问题阻挡。
- **版本过滤** — 更新检查跳过预发布版本（如 `v1_pr1`），仅检查稳定版 `vX.Y.Z`。
- **帮助页面扩展** — 新增「模型选择」「上下文用量」「撤销/重做」「检查更新」说明（中英双语）。
- **通知深层链接** — 点击通知直接导航到活跃的聊天会话。
- **子会话过滤** — 可选开关，在项目列表中隐藏子会话，仅显示根对话。
- **完整 API 文档** — `OPENCODE_API.md` 记录所有 REST 端点、SSE 事件和数据模型。

### 修复

- **流式消息闪烁/消失** — AI 回复完成后会短暂消失再重现。根因：占位项与最终消息的 LazyColumn key 不同，导致销毁+重建。改用统一占位方案，使用真实消息 ID，LazyColumn 就地更新内容，零视觉跳变。
- **流式输出卡住/阻塞** — 新增看门狗定时器检测并恢复 SSE 卡顿。`sendMessage()` 重入时中止陈旧状态，`initialize()` 在 App 重启时检测不完整的 assistant 消息。
- **模型选择未生效** — 服务端要求嵌套 `{ "model": { "providerID": "...", "modelID": "..." } }` 格式。修正 `SendMessageRequest` 使用 `ModelRef` DTO 包装模型选择。
- **上下文用量不准确** — 之前累加所有历史 token（重复计算）。现在通过反向扫描仅提取最新 assistant 消息的 `input` token，会话切换时正确重置。
- **session.idle 竞态条件** — `session.idle` 在消息加载完成前就清除了状态。新增延迟清除机制，带重试的消息获取和 assistant 消息存在性验证后才提交状态转换。
- **面板滑动手势冲突** — 手势检测与 LazyColumn 滚动冲突。改用累计拖拽距离阈值（40dp），从屏幕任意位置左滑即可打开面板。
- **右面板遮挡状态栏** — 增加 `WindowInsets.statusBars` 内边距。
- **国内更新检查失败** — 新增镜像源并发检查，至少一个源可达即可检测更新。
- **Gradle 守护进程挂起** — 在 `gradle.properties` 中禁用守护进程。

### 变更

- **聊天输入栏重新设计** — 三段式钢琴键风格：模型选择段 | 上下文用量指示 | 可展开文本输入 + 发送按钮。收起时紧凑，展开时功能完整。
- **SSE 管道优化** — EventBus 缓冲区 256（DROP_OLDEST）。Delta 事件 16ms 合并窗口。心跳超时 45s。所有角色的 `message.updated` 增量处理。
- **双通道刷新** — 3 秒消息轮询与 SSE 并行作为保底。会话列表页：SSE + 30 秒轮询。兜底轮询仅在 SSE 15 秒无事件时激活。
- **内存优化** — 消息列表上限 150 条。流式文本截断 10K 字符。
- **智能自动滚动** — 仅在用户已处于底部时自动滚动。向上翻看历史消息时不再被打断。
- **代码段默认展开** — 代码/文本段默认展开显示，避免"内容消失"的感觉。
- **面板动画** — `FastOutSlowInEasing(350ms)`，滑动更自然。
- 内部重构：`ChatUiState` 拆分为 `SessionMetaState`、`StreamingDisplayState`、`ChatDisplayState`。

## [1.1.2] - 2026-05-07

### 新增

- **后台 SSE 保活** — 前台 Service（`dataSync` 类型）在 App 进入后台时保持 SSE 连接不断开。持久通知「OConnector 正在运行」满足 Android 14+ 前台服务要求。
- **应用内更新检测** — 打开连接页面时自动检查 GitHub Releases 是否有新版本。有更新时，语言按钮旁边出现下载图标 (⬇)。
- **更新对话框** — 点击下载图标可查看更新日志和下载按钮。APK 通过 DownloadManager 下载，系统安装器完成安装。
- **帮助页面新增「检查更新」说明**（中英双语）。

### 修复

- **API 26-28 兼容性** — `ServiceInfo.FOREGROUND_SERVICE_TYPE_DATA_SYNC` 需要 API 29+。已添加 SDK 版本检查，旧设备自动回退到 2 参数 `startForeground()`。
- **更新 UI 国际化** — UpdateDialog 改用 `AppLocale` 本地化字符串，不再硬编码英文。

### 变更

- 帮助页面版本号改用 `BuildConfig.VERSION_NAME`，不再硬编码。
- APK 不再追踪在 git 仓库中 — 改为通过 GitHub Releases 分发。
- `release/` 目录已加入 `.gitignore`。

## [1.1.1] - 2026-05-07

### 新增

- **HTTPS (TLS) 支持** — 连接页面新增"使用 HTTPS (TLS)"开关。通过 HTTPS 反向代理（如 Lucky、Nginx、Caddy）连接时请开启此选项，APP 将使用 `https://` 替代 `http://` 进行所有 API 和 SSE 请求。
- **自签名证书支持** — 开启 TLS 后会显示"允许不受信任的证书"开关。连接使用自签名证书的服务器（家庭内网常见）时请开启。两项设置均会保存，重启 APP 后自动恢复。

### 修复

- **中文/非 ASCII 路径编码** — 从包含中文字符的目录加载会话时（如 `C:\Users\...\论文阅读`）会报错 `Unexpected char 0x8bba in x-opencode-directory`。现在 `x-opencode-directory` HTTP 头已正确进行 URL 编码，符合 RFC 7230（仅允许 ASCII 字符）。

## [1.1.0] - 2026-05-05

### 新增

- **多项目自动发现** — APP 现在会并行查询所有已知的 OpenCode 项目，合并展示每个项目的会话。无论 `opencode serve` 从哪个目录启动都能正常工作。

### 修复

- **暗色模式** — 之前完全无法使用（主题从未被应用到 Activity）。新增手动 🌙/☀️ 切换按钮，设置保存在 DataStore 中。
- **Todo 按钮可见性** — Todo 按钮在列表为空时隐藏。现在始终可见（有任务时显示角标，无任务时显示普通图标）。同时新增 `message.part.completed` SSE 事件处理，实现 Todo 实时更新。
- **工具调用渲染** — 连续的工具调用会相互覆盖，因为 `putSegment()` 仅按类型匹配。新增 `callID` 追踪，确保每个工具调用拥有独立的渲染段。
- **内存溢出崩溃** — 加载长会话时出现 108MB 内存分配崩溃。消息 API 新增 `?limit=50` 参数。移除了会导致 JSON 转义序列损坏的 `truncateLargeJsonStrings()`（此函数是消息/会话"消失"的根本原因）。
- **会话排序** — 会话按创建时间排序。现在改为按最后更新时间排序。
- **状态栏颜色** — 状态栏和导航栏颜色现在在亮色和暗色模式下都与界面颜色一致。
- **SSE 流式输出卡顿** — 流式输出有时会随机卡住不动，需要手动刷新。根因：`channelFlow` 中的 `trySend()` 在主线程繁忙时会静默丢弃 SSE 事件。已替换为 `send()` 提供正确的背压机制。同时将逐条日志改为每 50 条输出一次，减少主线程 JNI 开销。

### 变更

- 服务器启动脚本不再需要从特定目录启动。
- `listSessions()` 新增可选的 `scope` 参数用于服务端项目过滤。
